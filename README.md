# Todo List — Vue 3

Учебный проект на Vue 3 (Options API)

## Запуск

```bash
npm install   # установить зависимости
npm run dev   # запустить дев-сервер
```

После запуска открыть в браузере адрес из консоли (http://localhost:5173).

Сборка production-версии:

```bash
npm run build     # результат в папке dist/
npm run preview   # локальный просмотр собранной версии
```

## Функционал

- добавление задачи
- отображение списка задач
- отметка выполнения чекбоксом
- зачёркивание выполненной задачи
- удаление задачи

## Архитектура

```
src/
├── main.js                 — точка входа: только монтирование приложения
├── App.vue                 — корневой компонент: владеет состоянием (todos),
│                             связывает дочерние компоненты
├── services/
│   └── todoService.js      — бизнес-логика: чистые функции без Vue
└── components/
    ├── TodoForm.vue        — форма добавления (emit 'add')
    ├── TodoList.vue        — отрисовка списка (props: todos)
    └── TodoItem.vue        — одна задача (emit 'toggle' / 'remove')
```

## Принципы

**Single Responsibility** — каждый модуль делает одну вещь:

| Модуль | Ответственность |
|---|---|
| `main.js` | запуск приложения |
| `App.vue` | хранение состояния и связывание компонентов |
| `todoService.js` | операции над данными (создать, переключить, удалить) |
| `TodoForm.vue` | собрать текст от пользователя |
| `TodoList.vue` | отрисовать массив задач |
| `TodoItem.vue` | отобразить одну задачу |

**Разделение UI и бизнес-логики** — `todoService.js` не знает про Vue: это чистые функции, которые можно тестировать и переиспользовать отдельно от интерфейса. Компоненты только отображают данные и сообщают о действиях пользователя.

**Однонаправленный поток данных** — состояние живёт только в `App.vue`. Вниз данные передаются через props, вверх — через события (emit). Дочерние компоненты «глупые»: они не изменяют данные сами.

```
        props (данные вниз)
App.vue ──────────────────► TodoList ──► TodoItem
   ▲                                         │
   └───────── emit (события вверх) ◄─────────┘
```
